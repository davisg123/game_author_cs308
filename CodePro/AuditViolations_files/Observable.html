30 /**
31  * This class represents an observable object, or "data"
32  * in the model-view paradigm. It can be subclassed to represent an
33  * object that the application wants to have observed.
34  * <p>
35  * An observable object can have one or more observers. An observer
36  * may be any object that implements interface <tt>Observer</tt>. After an
37  * observable instance changes, an application calling the
38  * <code>Observable</code>'s <code>notifyObservers</code> method
39  * causes all of its observers to be notified of the change by a call
40  * to their <code>update</code> method.
41  * <p>
42  * The order in which notifications will be delivered is unspecified.
43  * The default implementation provided in the Observable class will
44  * notify Observers in the order in which they registered interest, but
45  * subclasses may change this order, use no guaranteed order, deliver
46  * notifications on separate threads, or may guarantee that their
47  * subclass follows this order, as they choose.
48  * <p>
49  * Note that this notification mechanism has nothing to do with threads
50  * and is completely separate from the <tt>wait</tt> and <tt>notify</tt>
51  * mechanism of class <tt>Object</tt>.
52  * <p>
53  * When an observable object is newly created, its set of observers is
54  * empty. Two observers are considered the same if and only if the
55  * <tt>equals</tt> method returns true for them.
56  *
57  * @author  Chris Warth
58  * @see     java.util.Observable#notifyObservers()
59  * @see     java.util.Observable#notifyObservers(java.lang.Object)
60  * @see     java.util.Observer
61  * @see     java.util.Observer#update(java.util.Observable, java.lang.Object)
62  * @since   JDK1.0
63  */

135         /*

145              * the Vector and store the state of the Observer
